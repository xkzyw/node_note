Indroduzione

Node.js è una framework di JavaScript open-source. È progettato per eseguire codice JavaScript lato server, consentendo agli sviluppatori di creare applicazioni web altamente scalabili e performanti.

A differenza dei tradizionali ambienti di esecuzione JavaScript, che sono tipicamente orientati al browser, Node.js consente di eseguire JavaScript sul server.

Endpoint

Uno dei feature principali offerto da Node.js, è la possibilità di creare degli endpoint che si riferisce a un particolare URL o URI (Uniform Resource Identifier) che è associato a una specifica azione o risorsa sul server.

Gli endpoint sono comunemente utilizzati nelle API (Application Programming Interface) per definire i punti di accesso a risorse o servizi.

Gli endpoint possono essere più complessi e associati a diversi metodi HTTP (come GET, POST, PUT, DELETE) per eseguire azioni specifiche o restituire dati specifici. La progettazione degli endpoint è un aspetto chiave nella creazione di un'API robusta e intuitiva, e può coinvolgere parametri, gestione degli errori, middleware, autenticazione e altro ancora, a seconda delle esigenze dell'applicazione.

Npm

npm (Node Package Manager) è uno strumento cruciale nell'ambiente di sviluppo di Node.js.

Progettato per semplificare la gestione delle dipendenze e la distribuzione di pacchetti di codice JavaScript, npm offre una serie di funzionalità essenziali.

Grazie a npm, si può definire e gestire le dipendenze dei propri progetti in modo strutturato attraverso il file package.json. Questo file contiene informazioni sul progetto e l'elenco dei pacchetti di terze parti necessari.

Si può generare il file package.json attraverso il commando fornito dal npm 

npm init
